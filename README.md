# Diary Records｜日记唱片

一个围绕手机 AI 助手与手机系统 AI 体验设计的可运行作品原型。它把照片、日记对话、情绪关键词、粒子影像、唱片视觉和私人音乐生成融合成适合移动端记录与分享的 AI 日记应用。

## 项目定位

Diary Records 的核心不是“帮用户写一篇日记”，而是把用户一天中的照片、想法、情绪和手机生态线索整理成一张可以回看、可以播放、可以分享的“日记唱片”。

产品面向年轻用户在学习、办公、出行、影像记录和社交分享中的真实需求：记录门槛低、情绪陪伴强、视觉表达高级、后续可与手机系统和其他应用协同。

## 已实现功能

- 创建某一天或某个时间段的日记卡片。
- 上传照片并生成可交互粒子影像。
- 支持日期、时间段、标题、一句话和心情标签。
- 支持与 AI 进行文字对话。
- 支持浏览器语音输入。
- 支持 AI 自动生成日记摘要、关键词、心情和标题。
- 支持粒子视图与唱片视图切换。
- 支持根据日记内容生成歌曲标题、歌词、副歌、曲风、BPM、调性和音乐 prompt。
- 支持本地 WebAudio demo 播放，音乐模型不可用时仍可完整演示。
- Memory 页面支持横向翻阅日记、打开详情、回看旧对话、播放唱片。
- Music 页面支持展示唱片、歌词、歌曲信息、导出封面图和系统分享。
- Info 页面包含产品介绍和手机生态 AI 联动模拟。
- 内置三篇完整样例日记：水晶创业收尾、第一只小猫、算法实习 offer。

## AI API 配置

点击右上角设置按钮，可配置 OpenAI-compatible 接口：

- `Base URL`：例如 `http://127.0.0.1:4000/v1`
- `API Key`：你的模型服务密钥
- `Chat / Vision Model`：用于日记对话、摘要、关键词提取和图片理解
- `Music Model`：用于音乐生成；当前原型会先尝试调用模型，失败后生成本地 demo

接口形式：

```http
POST /chat/completions
```

建议后续按能力拆分模型：

- 日记对话：中文表达自然、情绪陪伴稳定、支持长上下文。
- 图片理解：多模态视觉语言模型，用于识别照片场景、色彩、物体和情绪。
- 结构化摘要：支持 JSON 输出，用于标题、摘要、关键词、心情。
- 音乐生成：支持 song generation / music generation / audio generation。
- 语音输入：ASR 或 Whisper 类模型。
- 语音回复：TTS，让 AI 伙伴拥有固定声音人格。
